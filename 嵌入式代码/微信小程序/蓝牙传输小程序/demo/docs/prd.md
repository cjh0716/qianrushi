# 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
ESP32蓝牙配置小程序

### 1.2 产品定位
一款用于配置ESP32设备网络参数的微信小程序，通过蓝牙通信方式向ESP32设备下发城市、WiFi名称和WiFi密码等配置信息，实现设备的快速网络配置。本小程序是OneNET物联网监控系统的前置配网环节，设备通过蓝牙配置成功后，将通过WiFi接入OneNET平台进行数据上报和远程控制。

### 1.3 产品目标
- 为用户提供便捷的ESP32设备配置方式
- 简化设备配网流程，降低用户操作门槛
- 确保配置信息安全、准确地传输到设备
- 为后续设备接入OneNET物联网平台奠定基础

---

## 2. 问题陈述

### 2.1 当前现状
ESP32设备的网络配置通常需要通过串口连接或预先烧录配置文件，操作繁琐，对用户技术能力要求较高。

### 2.2 用户痛点
- 传统串口配置方式需要额外硬件（如USB转串口模块），设备携带不便
- 配置过程复杂，普通用户难以独立完成
- 配置信息容易因操作失误导致输入错误
- 缺乏可视化界面，配置状态反馈不直观

### 2.3 业务影响
- 设备配置环节成为用户使用门槛，影响产品整体体验
- 配置失败率高，增加用户支持成本
- 不利于产品的大规模推广和用户增长

---

## 3. 解决方案

### 3.1 方案概述
开发一款微信小程序，通过蓝牙通信方式，为ESP32设备提供可视化的配置界面。用户只需在小程序中输入配置信息，即可一键发送到设备，完成网络参数配置。

### 3.2 用户故事

**故事1：首次配置设备**
> 作为新用户，我希望打开小程序就能搜索并连接ESP32设备，输入WiFi信息后一键完成配置，让设备快速接入网络。

**故事2：修改网络配置**
> 作为已配置过设备的用户，我希望能够随时通过小程序重新连接设备，修改WiFi名称或密码，适应网络环境变化。

**故事3：确认配置结果**
> 作为用户，我希望在发送配置后能够看到配置是否成功，以及设备当前的连接状态，确保设备正常工作。

### 3.3 成功指标
- 设备配置成功率 ≥ 95%
- 用户完成配置平均时长 ≤ 1分钟
- 用户配置失败反馈率 ≤ 5%

---

## 4. 功能需求

### 4.1 核心功能

#### 功能1：蓝牙设备搜索与连接
- 小程序启动后自动搜索附近的蓝牙设备
- 显示可连接的蓝牙设备列表
- 支持手动触发重新搜索
- 点击设备列表项即可发起连接

#### 功能2：配置信息输入
- 提供城市输入框（string类型）
- 提供WiFi名称（SSID）输入框（string类型）
- 提供WiFi密码输入框（string类型，支持密码掩码显示）
- 输入内容实时校验（非空检查）

#### 功能3：配置信息发送
- 一键发送配置信息到已连接的ESP32设备
- 发送过程中显示进度状态
- 发送完成后显示结果反馈（成功/失败）

#### 功能4：连接状态管理
- 实时显示当前蓝牙连接状态
- 支持主动断开连接
- 连接异常时自动提示并尝试重连

### 4.2 辅助功能

#### 功能5：配置历史记录
- 保存用户最近的配置记录
- 支持快速选择历史配置信息
- 历史记录可手动删除

#### 功能6：使用引导
- 首次使用时展示简单的操作引导
- 关键步骤提供操作提示

---

## 5. 非功能需求

### 5.1 性能要求
- 蓝牙搜索响应时间 ≤ 3秒
- 设备连接建立时间 ≤ 5秒
- 配置信息发送时间 ≤ 2秒
- 小程序启动时间 ≤ 2秒

### 5.2 可靠性要求
- 蓝牙连接稳定性：连续工作24小时无异常断开
- 数据传输准确性：配置信息传输错误率 ≤ 0.1%
- 异常处理：网络异常、设备断开等场景有明确的错误提示和处理机制

### 5.3 安全性要求
- WiFi密码在本地输入和传输过程中进行加密处理
- 配置信息不存储在云端服务器
- 蓝牙通信需保证数据传输安全

### 5.4 兼容性要求
- 支持iOS 10.0及以上版本
- 支持Android 6.0及以上版本
- 支持微信版本 7.0及以上
- 兼容主流ESP32蓝牙模块

### 6.1 页面结构

#### 页面1：首页（设备搜索页）
- 页面标题：设备配置
- 搜索按钮/状态指示器
- 设备列表区域

### 6.3 视觉设计
- 采用简洁现代的设计风格
- 使用微信小程序标准组件样式

## 7. 数据需求

### 7.1 数据类型定义
| 数据项 | 类型 | 说明 |
| :--- | :--- | :--- |
| city | string | 城市名称，用于设备获取当地天气信息、时区配置或在OneNET平台展示设备所在位置 |
| ssid | string | WiFi网络名称 |
| password | string | WiFi网络密码 |

### 7.2 数据格式要求
- 所有数据项均为UTF-8编码字符串
- city最大长度：50字符
- ssid最大长度：32字符
- password最大长度：64字符

### 7.3 数据存储
- 配置历史记录存储在本地缓存中
- 最多保存10条历史记录
- 用户可手动清除历史记录

## 9. 风险与应对

### 9.1 技术风险

**风险1：蓝牙兼容性问题**
- 描述：不同品牌手机的蓝牙协议实现可能存在差异，导致连接不稳定
- 应对：在多种设备上进行兼容性测试，针对问题设备进行适配优化

**风险2：数据传输丢失**
- 描述：蓝牙传输过程中可能出现数据丢包
- 应对：确保数据传输的完整性，配置信息发送后需得到设备的确认反馈

### 9.2 业务风险

**风险1：用户操作失误**
- 描述：用户可能输入错误的WiFi信息导致配置失败
- 应对：增加输入校验和格式提示，提供配置结果反馈

**风险2：设备配对问题**
- 描述：首次配对可能需要用户在手机系统设置中确认
- 应对：提供清晰的操作引导，说明配对流程

### 9.3 合规风险

**风险1：蓝牙权限申请**
- 描述：小程序需要获取蓝牙权限，用户可能拒绝授权
- 应对：在首次使用时清晰说明权限用途，提供友好的授权引导
